1. Tipos de datos y declaración de variables

JavaScript es un lenguaje de tipado dinámico, lo que significa que no necesitas especificar el tipo de dato al declarar una variable. Los tipos de datos primitivos incluyen: number, string, boolean, undefined, null, symbol y bigint.

Para declarar variables puedes usar var, let o const. Se recomienda usar let para variables que pueden cambiar y const para constantes.

JavaScript convierte automáticamente los tipos cuando es necesario (coerción de tipos), pero es importante entender cómo funcionan para evitar errores comunes.

Ejemplos Prácticos:

Ejemplo 1: Declaración de variables con diferentes tipos

// Declaración de variables con diferentes tipos de datos
let nombre = "Juan";           // String
let edad = 25;                 // Number
let esEstudiante = true;       // Boolean
let salario = null;            // Null
let telefono;                  // Undefined
const PI = 3.14159;            // Constante numérica

console.log(typeof nombre);     // "string"
console.log(typeof edad);       // "number"
console.log(typeof esEstudiante); // "boolean"

Ejemplo 2: Conversión de tipos

// Conversión explícita de tipos
let numeroTexto = "123";
let numero = Number(numeroTexto);    // Convierte string a number
let textoNumero = String(456);       // Convierte number a string
let booleano = Boolean(1);           // Convierte a boolean

// Conversión implícita (coerción)
let suma = "5" + 3;                  // "53" (concatenación)
let resta = "5" - 3;                 // 2 (conversión a número)
let comparacion = "5" == 5;          // true (conversión automática)

2. Operadores y expresiones

Los operadores en JavaScript permiten realizar operaciones sobre variables y valores. Existen varios tipos: aritméticos (+, -, *, /, %), de comparación (==, ===, !=, !==, <, >), lógicos (&&, ||, !), y de asignación (=, +=, -=, etc.).

Es importante distinguir entre == (comparación con conversión de tipos) y === (comparación estricta sin conversión). Los operadores lógicos utilizan evaluación de cortocircuito, lo que significa que pueden no evaluar todos los operandos.

La precedencia de operadores determina el orden en que se evalúan las expresiones. Los paréntesis pueden usarse para cambiar este orden y hacer el código más legible.

Ejemplos Prácticos:

Ejemplo 1: Operadores aritméticos y de asignación

// Operadores aritméticos básicos
let a = 10;
let b = 3;

let suma = a + b;           // 13
let resta = a - b;          // 7
let multiplicacion = a * b; // 30
let division = a / b;       // 3.3333...
let modulo = a % b;         // 1 (resto de la división)

// Operadores de asignación
let contador = 5;
contador += 3;              // contador = contador + 3 (8)
contador -= 2;              // contador = contador - 2 (6)
contador *= 2;              // contador = contador * 2 (12)

// Incremento y decremento
let x = 5;
console.log(x++);           // 5 (post-incremento)
console.log(++x);           // 7 (pre-incremento)

Ejemplo 2: Operadores de comparación

// Comparaciones con == vs ===
console.log(5 == "5");      // true (conversión automática)
console.log(5 === "5");     // false (comparación estricta)
console.log(null == undefined);  // true
console.log(null === undefined); // false

// Otros operadores de comparación
let edad = 18;
console.log(edad >= 18);    // true
console.log(edad !== 17);   // true

// Comparación de strings
console.log("apple" < "banana");  // true (orden alfabético)
console.log("10" < "9");          // true (comparación de strings)

Ejemplo 3: Operadores lógicos y precedencia

// Operadores lógicos
let esMayor = true;
let tieneID = false;

console.log(esMayor && tieneID);    // false (AND lógico)
console.log(esMayor || tieneID);    // true (OR lógico)
console.log(!tieneID);              // true (NOT lógico)

// Evaluación de cortocircuito
let nombre = "" || "Usuario Anónimo";  // "Usuario Anónimo"
let saludo = nombre && "Hola " + nombre; // "Hola Usuario Anónimo"

// Precedencia de operadores
let resultado = 2 + 3 * 4;          // 14 (no 20)
let resultado2 = (2 + 3) * 4;       // 20 (con paréntesis)

3. Entrada y salida de datos

En JavaScript, la entrada y salida de datos puede realizarse de varias formas. En el navegador, podemos usar prompt() para entrada, alert() y console.log() para salida. También podemos interactuar con elementos HTML usando el DOM.

Para aplicaciones web modernas, es más común usar formularios HTML y eventos para capturar la entrada del usuario, y manipular el DOM para mostrar resultados. console.log() es especialmente útil para depuración y desarrollo.

Es importante validar siempre los datos de entrada del usuario, ya que pueden introducir valores inesperados o maliciosos. JavaScript proporciona varios métodos para convertir y validar datos.

4. Manipulación de cadenas y números

JavaScript proporciona muchos métodos integrados para manipular strings y números. Los strings son inmutables, lo que significa que los métodos no modifican el string original sino que devuelven uno nuevo.

Los métodos más comunes para strings incluyen: length, charAt(), substring(), indexOf(), replace(), split(), toUpperCase(), toLowerCase(), y trim().

Para números, JavaScript ofrece métodos como toFixed(), parseInt(), parseFloat(), y el objeto Math con funciones matemáticas avanzadas como Math.round(), Math.random(), y Math.max().

Ejemplos Prácticos:

Ejemplo 1: Manipulación de strings

// Propiedades y métodos básicos de strings

console.log("Sin espacios:", frase.trim());                // "Hola Mundo JavaScript"
console.log("Mayúsculas:", frase.toUpperCase());           
console.log("Minúsculas:", frase.toLowerCase());

// Búsqueda y extracción
console.log("Posición de 'Mundo':", frase.indexOf("Mundo")); // 7
console.log("Incluye 'Java':", frase.includes("Java"));       // true
console.log("Substring:", frase.substring(2, 6));            // "Hola"

// Reemplazo y división
let nuevaFrase = frase.replace("JavaScript", "JS");
let palabras = frase.trim().split(" ");
console.log("Reemplazado:", nuevaFrase);
console.log("Palabras:", palabras);

Ejemplo 2: Manipulación de números

// Función para formatear nombres
function formatearNombre(nombre) {
    return nombre.trim()
                 .toLowerCase()
                 .split(' ')
                 .map(palabra => palabra.charAt(0).toUpperCase() + palabra.slice(1))
                 .join(' ');
}

// Función para formatear números como moneda
function formatearMoneda(cantidad) {
    return "$" + cantidad.toFixed(2).replace(/\d(?=(\d{3})+\.)/g, '$&,');
}

// Validación de formato
function validarFormato(texto, tipo) {
    const patrones = {
        email: /^[^\s@]+@[^\s@]+\.[^\s@]+$/,
        telefono: /^\d{3}-\d{3}-\d{4}$/,
        codigo: /^[A-Z]{2}\d{4}$/
    };
    
    return patrones[tipo] ? patrones[tipo].test(texto) : false;
}

// Ejemplos de uso

console.log("Moneda:", formatearMoneda(1234567.89));
console.log("Email válido:", validarFormato("test@email.com", "email"));

5. Template literals e interpolación

Los template literals (literales plantilla) son una característica de ES6 que permite crear strings de forma más flexible usando backticks (`). Permiten interpolación de variables con ${}, strings multilínea sin caracteres de escape, y llamadas a funciones dentro del string.

La interpolación hace que el código sea más legible comparado con la concatenación tradicional. También permiten expresiones JavaScript complejas dentro de las llaves, incluyendo operadores ternarios y llamadas a funciones.

Los template literals también soportan "tagged templates", una característica avanzada que permite procesar el template con una función personalizada, útil para internacionalización, sanitización, o formateo especial.

Ejemplos Prácticos:

Ejemplo 1: Interpolación básica

// Comparación: concatenación vs template literals
let nombre = "Carlos";
let edad = 30;
let profesion = "Desarrollador";

// Forma tradicional (concatenación)
let mensaje1 = "Hola, mi nombre es " + nombre + 
              ", tengo " + edad + " años y soy " + profesion + ".";

// Con template literals
let mensaje2 = `Hola, mi nombre es ${nombre}, tengo ${edad} años y soy ${profesion}.`;

console.log("Concatenación:", mensaje1);
console.log("Template literal:", mensaje2);

// Expresiones dentro de template literals
let precio = 99.99;
let descuento = 0.15;
let mensaje3 = `El precio final es: $${(precio * (1 - descuento)).toFixed(2)}`;
console.log(mensaje3);

6. Estructuras condicionales

Las estructuras condicionales permiten que el programa tome diferentes caminos según se cumplan o no ciertas condiciones. JavaScript ofrece varias formas: if...else, switch, y el operador ternario ? :.

La estructura if...else es la más básica y versátil. Puede encadenarse con else if para múltiples condiciones. El switch es útil cuando se compara una variable contra múltiples valores específicos.

El operador ternario es una forma compacta de escribir condiciones simples. También existen operadores de cortocircuito (&&, ||) que pueden usarse para ejecución condicional. Es importante entender los valores "falsy" en JavaScript: false, 0, "", null, undefined, NaN.

Las condiciones anidadas pueden volverse complejas, por lo que es importante mantener el código legible y considerar refactoring cuando sea necesario.

Ejemplos Prácticos:

Ejemplo 1: Estructuras if...else básicas y anidadas

// Estructura if...else básica
function evaluarNota(nota) {
    let resultado;
    
    if (nota >= 90) {
        resultado = "Excelente";
    } else if (nota >= 80) {
        resultado = "Muy bueno";
    } else if (nota >= 70) {
        resultado = "Bueno";
    } else if (nota >= 60) {
        resultado = "Suficiente";
    } else {
        resultado = "Insuficiente";
    }
    
    return resultado;
}

// Condiciones anidadas
function verificarAcceso(edad, tieneID, esEstudiante) {
    if (edad >= 18) {
        if (tieneID) {
            return "Acceso completo";
        } else {
            return "Necesita identificación";
        }
    } else {
        if (esEstudiante) {
            return "Acceso con supervisión";
        } else {
            return "Acceso denegado";
        }
    }
}

// Ejemplos de uso
console.log("Nota 85:", evaluarNota(85));
console.log("Acceso (17, false, true):", verificarAcceso(17, false, true));
console.log("Acceso (20, true, false):", verificarAcceso(20, true, false));

7. Bucles

Los bucles permiten ejecutar un bloque de código repetidamente mientras se cumpla una condición. JavaScript ofrece varios tipos de bucles: for, while, do...while, y el bucle for...in para iterar sobre propiedades de objetos, y for...of para iterar sobre elementos de colecciones iterables como arrays.

El bucle for es el más común y versátil, permitiendo definir un contador, una condición de continuación, y un incremento. El bucle while ejecuta el bloque mientras la condición sea verdadera, y el do...while garantiza al menos una ejecución del bloque antes de evaluar la condición.

Es importante tener cuidado con los bucles infinitos, que ocurren cuando la condición nunca se vuelve falsa. También existen técnicas como el "break" para salir del bucle anticipadamente y "continue" para saltar a la siguiente iteración.

Ejemplos Prácticos:

Ejemplo 1: Bucle for básico

// Bucle for básico
for (let i = 0; i < 5; i++) {
    console.log("Iteración:", i);
}

Ejemplo 2: Bucle while

// Bucle while
let i = 0;
while (i < 5) {
    console.log("Iteración:", i);
    i++;
}

Ejemplo 3: Bucle do...while

// Bucle do...while
let i = 0;
do {
    console.log("Iteración:", i);
    i++;
} while (i < 5);

Ejemplo 4: Bucle for...of

// Bucle for...of
